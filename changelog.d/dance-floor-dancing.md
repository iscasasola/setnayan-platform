## 2026-07-09 · fix(plan3d): walking off the dance floor no longer teleport-pops on frame one

Tap the dance floor and the roaming figure walks on (via `danceObstacles`, which drops the dance-floor avoidance disc so it can reach the floor) and dances. That dance walk clamps against `danceObstacles`, so `walkerPosRef` comes to rest INSIDE the dance-floor disc. Tapping any non-dance point then re-used `roamObstacles` — which re-includes that disc — so `steerPath`'s `path[0] === from` was still inside it and the first per-frame `pushOutOfDiscs` clamp snapped the figure to the disc edge in a single frame: a visible ~0.6–2.6 m teleport pop outward on the first frame of every walk off the dance floor. (Review finding, CONFIRMED. The guest venue surface was never affected — `GuestAvatar` has no per-frame clamp.)

- `plan3d-scene.tsx` (`walkToPoint`): before pathing, drop exactly the obstacle disc(s) that CONTAIN the start point `from` (`dropDiscsContaining(..., AVATAR_BODY_R)`) from both the steer path and the per-frame clamp set. Today the only live case is leaving the dance floor, but the relief is general — it mirrors the seat walks' existing arrive-point drop, applied at the START. When nothing contains `from` the original obstacle set passes through untouched (no per-tap grid rebuild). Fixes the reported floor-tap walk-off and the booth walk-off (`handleWalkToBooth`), which share `walkToPoint`.
- `plan3d-scene.tsx` (`handleSeatTap`): tapping your own seat FROM the dance floor hit the same class of bug (its path + clamp sets kept the dance disc while `from` sat inside it → contorted exit + frame-one pop). Same from-disc drop now rides along on both its `pathObstacles` and `clampObstacles` (the latter still also drops the arrive-point neighbour discs).

Dancing still clears correctly when you walk away (a non-dance tap sets `dance=false`), the dance floor stays reachable, and reduced motion is unchanged (it teleports with `obstacles: []`, so it never clamped and never popped). No change to ordinary roam, seat/booth reachability, or the chase cam.
