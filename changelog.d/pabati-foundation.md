## 2026-06-22 · feat(pabati): video-guestbook foundation — schema + guest-clip upload pipeline

Conflict-free GROUNDWORK for PABATI, the guest video-greeting collector (up to 300 guests-or-the-couple record a ≤5s video greeting into a shared, couple-reviewed gallery on the event landing page). Schema + backend ONLY — the collector UI, the landing-page surface, and the recap/day-of wiring land in a later "surfaces" PR. The v2 catalog deliberately stays `PABATI: 'not_built'` so a couple can't buy a feature with no surface.

- **Migration `20270214296222_pabati_video_guestbook.sql`** (allocator-assigned non-round prefix, later than all on `main`) — mirrors the canonical Papic pattern (`20260718000000_papic_guest_seats_provisioning.sql`):
  - `public.pabati_clips` (IF NOT EXISTS) — `clip_id` UUID, `event_id` (CASCADE), nullable `guest_id` (SET NULL — a deleted guest's greeting survives; un-identified public prompts have none), `r2_object_key`, `duration_ms`, `guest_label`, `moderation_state` default `'unscreened'`, `captured_at`, `hidden_at`, `created_at`. Indexes on `event_id` + `captured_at`. **RLS at CREATE TABLE time** — `pabati_clips_couple_read` (is_admin() OR `event_members` member_type='couple') + `pabati_clips_admin_all`. No anon/public table policy (guest writes go through the RPC).
  - `public.pabati_event_owns_pabati(event)` — **bundle-aware + ADMIN-APPROVED** gate: TRUE iff an order/bundle confers PABATI AND its status is `paid`/`fulfilled`. Reuses the live `public.bundles_granting_sku()` helper (`20270103010000`) for the MEDIA_PACK fan-out; matches `lib/entitlements.ts` `checkOrderActive` (a still-pending `submitted` order does NOT confer access — this is a feature gate).
  - `public.pabati_record_clip(event, guest?, r2_key?, duration_ms?, guest_label?)` — SECURITY DEFINER, advisory-locked PER EVENT. Ownership check → `not_owned`; count vs **300 cap** → `quota_exhausted`; else INSERT with **`LEAST(duration_ms, 5000)`** (5s hard cap, server-side, can never store longer). Returns `{status, total, used, remaining}`. GRANT EXECUTE to authenticated + anon.
- **`apps/web/lib/pabati.ts`** — `eventPabatiActive()` = `eventSkuActive('PABATI')` (the handshake gate); `fetchPabatiQuota()` mirrors Papic's `fetchGuestQuota` (per-EVENT count vs the 300 cap, graceful-degrade to full-remaining on 42P01/42703 so it's safe pre-migration); `screenPabatiClipPoster()` drives the NSFW screen over the clip's poster frame via the shared `lib/nsfw-screen` classifier (fail-open).
- **`apps/web/app/api/pabati/clip/route.ts`** — mirrors `/api/papic/guest-capture`, photo→video: identifies via `readGuestSession()` OR an authenticated couple/coordinator (`event_members`); validates `file` is a `video/*` ≤25MB; **enforces the 5s cap server-side** (rejects a client-stamped `duration_ms > 5000`); quota pre-check (avoid R2 orphans); `r2Upload` to `R2_BUCKETS.media` under `pabati/<eventId>/`; records via `pabati_record_clip`; **NSFW screen in an `after()` block** over the client-extracted poster; best-effort Google-Drive copy (`artifactType: 'pabati'`).
- **`apps/web/lib/offline/service-handlers/pabati-handler.ts`** — replaced the `{ok:false,'V1.x post-pilot'}` stub: `syncOne` reconstructs the multipart body from the IndexedDB payload, extracts the poster at drain time, and POSTs to `/api/pabati/clip` (mirrors the camera-bridge handler, the one real non-stub handler).

Corpus hard locks honored: **5-second clip cap** (client-stamped value rejected at the route + clamped in the RPC), **NSFW screen on by default / cannot disable** (no toggle anywhere), **max 300 clips/event** (advisory-locked count gate). DO-NOT-TOUCH respected: no edits to `[slug]/page.tsx`, the recap page, the day-of card grid, or `video-guestbook-card.tsx`; catalog PABATI stays `'not_built'`.

NSFW-screening deviation (surfaced): the shared `screenCapture()` engine is image-only and its `ScreenCaptureTable` type is `papic_*`-only, so Pabati does NOT reuse it directly — instead `screenPabatiClipPoster()` calls the shared `classifyImageBytes`/`decideNsfw` over a poster JPEG (the clip's screening proxy, exactly how `screenCapture` screens a Papic clip; nsfwjs can't read video and the lambda has no ffmpeg). A posterless clip stays `'unscreened'` and is structurally excluded from guest surfaces — same behavior as a posterless Papic clip.
