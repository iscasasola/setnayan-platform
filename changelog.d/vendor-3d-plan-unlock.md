## 2026-07-22 · feat(vendor): 3D Plan unlock — vendor-enabled couple discount (₱1,000 SEATING_3D)

Ships the deferred activations half of the vendor 3D Booth add-on (owner-approved 2026-07-22). A booked vendor with an ACTIVE paid 3D Booth add-on (₱1,500/28d · `vendor_profiles.booth_addon_expires_at` · `isVendor3dBoothActive`) can UNLOCK the 3D Plan upgrade for a couple they're booked with — unlimited times, free (the ₱1,500 add-on is the charge). Unlocking does NOT gift `SEATING_3D` for free: it marks the event ELIGIBLE for a DISCOUNTED **₱1,000** `SEATING_3D` (vs the standard ₱2,999 catalog price). The COUPLE then buys it via the normal apply-then-pay checkout, and the server-authoritative price resolver charges ₱1,000. Both sides pay; the couple keeps full control of what they publish.

- **Migration `20270909783681_vendor_3d_plan_unlock_by_vendor.sql`** — new table `event_vendor_3d_plan_unlocks` (per-event discount-eligibility + attribution). `UNIQUE(event_id)` makes the unlock idempotent per event (first booked vendor wins the attribution; a second unlock no-ops). RLS at CREATE TABLE: the paying vendor reads its own unlocks (`current_vendor_profile_ids()`), the couple/coordinator reads their event's row (for the acknowledgement), admin `is_admin()` full. No write policy — writes come only from the vendor server action via the admin client. This is NOT a parallel entitlement: it feeds no `eventSkuActive()` and confers no free access.
