- **`supabase/migrations/20270728200000_flatten_inquiry_burn_to_1_token.sql`** — flattens the inquiry-unlock burn to a constant **1 token** for every region. The AUTHORITATIVE lever is `public.regions.burn_band`: both the live burn RPC `unlock_vendor_event` (20270401611377) and the hold RPC `unlock_vendor_event_hold` (20270726988829) resolve `events.region → regions.burn_band` and set `v_tokens := v_band`, so `UPDATE regions SET burn_band = 1` makes every unlock charge — and every HOLD reserve — exactly 1 token (= ₱200). The deprecated/read-free `token_burn_bands` table is ALSO synced to `tokens=1,band=1` (guarded by `to_regclass`) purely for hygiene — it drives no charge. Region banding was reconciled onto `regions.burn_band` at 20270331100000; the DECISION_LOG's "flatten token_burn_bands" wording predates that single-source move. No feature flag, tier gate, or RPC body is altered.

Net effect once applied: every vendor lead-unlock costs a uniform 1 token = ₱200 nationwide; the token HOLD-and-release mechanic (PR #3133/#3134) holds/charges 1 token. Nothing changes until `supabase db push` is run by the owner.
