## 2026-07-08 · chore(mood-board): palette-v2 review fixes — role-chip dot sees split keys, honest compat comment

Post-merge review of the palette-taxonomy-v2 landing (split wedding-party keys + parents + Nikah principals + room-dressing overrides). Two confirmed cheap fixes; one finding skipped as intended behavior.

- **Guest-list RoleChip color dot now mirrors the 3D attire resolver.** The dot resolved via `roleGroupOf(role)` → the coarse group key only, so a couple who filled a split sub-key (e.g. `bridesmaids`) but left the shared `wedding_party` fallback empty saw NO dot even though the 3D avatar was painted that color. RoleChip now reads `getPrimaryColor(palette, paletteKeyForRole(role)) ?? getPrimaryColor(palette, group)` — the guest's specific key first, then the group fallback — so the chip and the scene agree. Bonus: bride/groom chips now carry their own attire dot (the `couple` group has no aggregate primary).
- **Corrected an overclaiming compat comment** in `seating/lab/page.tsx`: a `wedding_party`-only palette matches the old GOWN bucket (`wedding_party ?? bride`), but suit-class attire — old code took groom/charcoal — now also degrades to `wedding_party` per the owner-locked v2 intent (see `mood-board.test.ts` "wedding_party-only dresses gowns AND suits identically"). The comment claimed "identical … old gown/suit buckets" for both; only the gown bucket is truly identical.
- **Skipped (intended, not a regression):** the observation that a bride/groom-only palette resolves entourage attire by SIDE (via `sideAttireColor`, preferring bride for `both`) rather than by garment. This is the deliberate, unit-tested v2 fallback (step 3 of the STRICT chain); the only backward-compat guarantee the PR makes — and tests as NON-NEGOTIABLE — is the `wedding_party`-only path, which is preserved byte-for-byte. Surfaced to owner: a suit/barong-wearing `both`/null-side guest (e.g. an officiant) now shows the bride color under a bride/groom-only palette — a design consequence of the side-based fallback, changeable if undesired.
