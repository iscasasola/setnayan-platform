## 2026-06-22 · feat(led): recolour the LED Background from the couple's Mood Board palette

The 0005 LED Background Maker now recolours its template gradient FROM the couple's Mood Board (`events.role_palette`), so the venue's stage wall reads as THEIR wedding — in lockstep with the Save-the-Date reveal + branded QR, which already pull from that same producer. Before this, the LED recoloured only from each template's hardcoded `[bg, accent1, accent2]` arrays and never read the Mood Board, breaking the branding flywheel.

- **`lib/site-palette.ts`** — new `ledPaletteFromMoodBoard(palette, templatePalette)` helper. Maps the de-duped Mood-Board pool onto the template's three gradient slots without flattening the template's character: `accent1` ← the boldest (most colourful) swatch; `accent2` ← the next visibly-distinct swatch, or a lightened lift of the bold one for single-hue palettes (so the two radial blooms stay separable); `bg` ← the template's OWN bg tinted toward the couple's deepest (dark templates) or lightest (light templates) swatch — a dark monogram-on-black template keeps a dark base, a light one keeps light. Decor, not legible UI → no WCAG floor (the LED wall carries no body text). Returns `null` when the palette is empty / colourless → caller keeps the template's hardcoded palette (existing look).
- **`studio/led/page.tsx`** — also selects `role_palette`, sanitizes via the shared `sanitizeRolePalette` (the helper the sibling studios use), passes it into `OwnedEditor` → `LedBackgroundMaker`.
- **`studio/led/_components/led-background-maker.tsx`** — every template thumbnail in the gallery + the selected template's customizing panel recolour from the Mood Board; a "Recoloured from your Mood Board" cue + 3-swatch chip surfaces when a board exists, so it feels intentional. The render payload carries the resolved palette. Template picker + loop/photo-pool controls unchanged.
- **`api/led-background/save/route.ts`** — re-resolves the palette server-side (authoritative — does NOT trust the client's `palette`) from `events.role_palette` and writes the resolved `[bg, accent1, accent2]` into `config_json.palette`, so the saved/rendered LED uses the couple's colours instead of deferring to a not-built template `defaults.json`. Graceful-degrades to the template default on a missing column (42703).
- **`lib/site-palette.test.ts`** — 5 new `ledPaletteFromMoodBoard` cases (null fallback for empty/colourless palettes, boldest→accent1 mapping, dark/light tone preservation, single-hue separability).

**Fallback behaviour:** a couple with no Mood Board (empty `role_palette`) sees the EXACT existing look — every template keeps its hardcoded palette end-to-end (editor preview, save, render). No visual change for them.

SPEC IMPACT: 0005 LED / 0010 mood board — the LED Background Maker now recolours from the couple's Mood Board palette like the other branded surfaces (Save-the-Date reveal, branded QR). No SKU / pricing change.
