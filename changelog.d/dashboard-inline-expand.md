## 2026-07-13 · feat(dashboard): "Around your event" cards expand IN PLACE — no whole-screen reload

Owner directive 2026-07-13: "when some cells are clicked on the dashboard, it tends to load the whole screen instead of it expanding the place where it should only load … we just want isolated loading. only those that needs to be loaded." This reasserts the owner's original "does not expand and collapse" note on the couple Event Overview doorstep band and **reverses the whole-card stretched-link doorway** shipped in [PR #3188](https://github.com/iscasasola/setnayan-platform/pull/3188) (the Event Overview council's "depth = navigation, no accordion" conclusion). Under the old pattern, tapping a doorstep card navigated to its full section route — a `force-dynamic` page behind a full-screen `loading.tsx` skeleton that re-ran ~10+ Supabase queries — which is exactly the "loads the whole screen" the owner was seeing.

- **New primitive `app/dashboard/[eventId]/_components/expand-card.tsx`** — a client `<ExpandCard>` that toggles open/closed as pure client state over content the parent **already server-rendered** (handed in as an RSC child). Expanding mounts content that is already in the flight payload: NO route change, NO full-page skeleton, NO re-fetch — the card's data was loaded once for its collapsed summary and is never loaded "again and again." The old whole-card `after:absolute after:inset-0` stretched `<Link>` is gone; the full-section editor stays reachable as a SECONDARY link at the card foot, so deep work still has a doorway — it is just no longer the only, screen-reloading option. WAI-ARIA accordion semantics (heading-wraps-button, `aria-expanded` + `aria-controls`); empty cards (nothing to reveal) render static with no chevron.
- **`event-dashboard.tsx`** — the four list-bearing doorstep cards (Hosts · Your team · Your services · Schedule) now render via `<ExpandCard>`: a substantive one-line summary while collapsed, the FULL list in place when expanded (no more top-4 slice + dead "+N more on the X tab" text). The summary-only **Conversations** card keeps its explicit "Open threads →" link but drops the whole-card stretched overlay, so an errant body tap no longer navigates. The band header, global vendor-masking footnote, and "See all recent activity →" entry are unchanged.

Verified against the real `<ExpandCard>` component in an isolated dev harness (client-only, no auth): expand reveals all rows in place, collapse restores the summary, `location.pathname` never changes, toggle buttons carry accessible names, and the console is clean. `tsc --noEmit` green.

SCOPE: this PR converts the Event Overview doorstep band only — the reference implementation of the pattern. Rolling it to the Services/Studio grid, the my-events launcher, and heavier sections (via an in-place drawer for full editors) is a follow-up, pending owner confirmation of the reversal.
