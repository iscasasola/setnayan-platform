## 2026-07-12 · feat(ai): the Event Brief specialty layer + stop dropping per-type signature answers

Each event type has its own signature specialty (weddings = love story, debut = the 18, christening = godparents, birthday = milestone …). This lands the deterministic (Rule 1) foundation for it — no new onboarding UI, no LLM.

- **Schema:** `supabase/migrations/…_events_signature_details_per_type_payload.sql` adds nullable `events.signature_details JSONB` — the generalised per-type signature store. Idempotent, reversible, no RLS change (row-level policies are column-agnostic). Name avoids the `event_details` invitation-widget-type collision and the vendor `specialty` taxonomy.
- **Persistence (fixes a silent drop):** the generic `/onboarding/[type]` flow already *collects* per-type signature answers (`details`) but discarded them at commit. Now persisted — `GenericOnboardingPayload.signatureDetails?` (types.ts) → `buildGenericEventInsert` writes `signature_details` (the map, or NULL when empty) → one-line wire `signatureDetails: details` in `generic-onboarding.tsx`. Empty → NULL so the Brief reads "not captured" honestly. Wedding + simple commit paths untouched.
- **Brief:** `lib/event-brief.ts` gains a 5th `specialty` layer — `SPECIALTY_KIND_BY_TYPE` (kind derived from `event_type`, pure/DB-free), reads `signature_details`, and a wedding surfaces its `love_story` as its specialty (love_story stays the canonical wedding store; the Brief aliases it). `computeRichness` is now applicability-aware: the specialty weight replaces the old story check and only counts for types that declare a specialty, so `simple_event` (and uncatalogued admin types) are never penalised for lacking one.
- **Tests:** +6 Brief tests (specialty aliasing, empty love_story, generic read, JSON-string variance, simple_event non-penalty, applicable-but-absent) and +3 persistence tests. `tsx --test`: **21/21 pass**. Shipped-#3135 invariants preserved (wedding 1.0, thin 0.1, empty 0). Adversarially reviewed (schema/persistence/brief/regression) — zero confirmed findings.
