## 2026-07-13 · feat(launcher): events timeline + expand/collapse home (no more shuttling off-page)

Owner directive (2026-07-13): on the account launcher (`/dashboard`), "everything on the home page … must expand and collapse on the page" and "not open a new page" — EXCEPT the three role-routed dashboards (event, vendor, admin), which still navigate. Plus: "your events should make it look like a timeline."

- **Your events → a Facebook-style TIMELINE.** The horizontal card scroll is replaced by a vertical spine (`<ol>`): one date-stamped node per event (colored dot on a connecting line), ordered date DESCENDING — **newest on top, older as you scroll down** — with undated ("Date to be set") events at the tail of the upcoming block, ending in a terminal "New event" node. **By default only UPCOMING + ongoing (today) events show; COMPLETED (past) + archived are hidden** behind "Show all" (`?show=all`) and continue oldest-toward-the-bottom. Each node is a compact horizontal card carrying the same signals as the old card (badge · real monogram · place/date · gold progress ring · countdown · attention line) and still deep-links into the event dashboard (an allowed jump). New components: `TimelineRow` + `TimelineNewEvent` (replacing `EventCard`/`NewEventCard`).
- **Your account → expand/collapse rows.** People · Memories Hub · Setnayan AI are no longer link tiles; each is an `<Expandable>` accordion that opens its content INLINE. People and Setnayan AI render prod-accurate inline bodies (coming-soon previews mirroring their pages); Memories Hub streams the real cross-event `PhotosTab` behind `<Suspense>`. New reusable client component `_components/expandable.tsx` (grid-rows height animation, `inert`/`aria-hidden` while collapsed) + server bodies in `_components/account-inline.tsx`. `AccountTile` removed.
- **Your spaces.** The vendor shop(s) + admin HQ cards still NAVIGATE (their own dashboards — the only allowed jumps). The flag-off "Life Story" doorway (which used to link to `/dashboard/library`) now expands INLINE via `<Expandable>` + `LifeStoryInline`. The section hides entirely when it would be empty.
- **Your year.** The strip no longer links out to `/dashboard/year`: it shows the first 3 derived moments and expands the rest INLINE via a new client `_components/year-moments-list.tsx` ("See your year · N more" ↔ "Show less"). Event-anchored moments still deep-link into their dashboards; derived/undated moments are plain non-navigating rows. `year-moments-strip.tsx` now precomputes Asia/Manila display strings server-side and hands them to the client list.
- "New event" stays a navigation (create-event is a flow, not a page of content to preview) and sits as the end-of-spine node.
- **Your year — monthsaries (owner 2026-07-13, PH monthly-anniversary culture).** A "together since" (relationship-origin) anchor now also surfaces its **next monthsary** as a single quiet line ("Your 29th monthsary"), the monthly sibling of the yearly anniversary. New `nextMonthsary()` in `lib/event-anchor.ts` (whole-months count, on-or-after, day-of-month clamp) + a `monthsary` kind + a relationship-only branch in `lib/year-moments.ts`. Deliberately restrained: only the SINGLE next monthsary shows (never a monthly stream), it stays a quiet non-milestone line, and the 12/24-month marks are SKIPPED (those land on the anniversary's date, so the anniversary label wins — no double-up). Wedding-origin anchors get no monthsary. +6 unit tests across `event-anchor.test.ts` (35/35) and `year-moments.test.ts` (19/19).
- **Your year — moment labels name their KIND, and COUNT the age.** Wedding anniversaries already read "Your Nth wedding anniversary" (unchanged; the type is identified). Recurring `birthday` moments now self-identify AND carry the ordinal age: with a birth anchor (`anchor_date`) the line reads "Lolo Ramon — 70th birthday" (a pet reads "Rocky — 5th birthday"); with no anchor it degrades to a plain "{name} — birthday" (skipped when the title already says "birthday"/"bday"). New `birthdayLabel()` + dedicated birthday branch in `lib/year-moments.ts`. Rationale (owner 2026-07-13): "dates only help to measure time gaps … since these dependents can be dogs, toy, sentimental items, lolo Ramon, is safe to count" — the age is a pure COUNT off a date the user already stored, and does NOT relax the birthdate-STORAGE fence / minor-consent gate in `lib/dependent-people.ts` (owner-locked + counsel-pending, unchanged). Age today only shows where a birth anchor is present; the event create-flow doesn't set one yet, so it's forward-ready for the (flag-gated) dependent-birthday feed. +4 unit tests (`year-moments.test.ts`, 16/16 green).
