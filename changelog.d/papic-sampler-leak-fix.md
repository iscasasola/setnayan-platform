## 2026-06-25 ¬∑ fix(papic): close the free-sampler storage leak ‚Äî record-layer cap + anon rate limit (+ R2 lifecycle as owner action)

Owner-flagged "FIX before widening." The free Papic sampler (3 seats ¬∑ 8 photos + 2 clips each ¬∑ 30-day retention ¬∑ login-free) capped captures ONLY in the `recordSeatCapture` server action ‚Äî i.e. AFTER the client had already presigned `/api/upload` and PUT the bytes to R2. A scripted (or just-leaked-link) claimer could presign the 9th/10th shot, PUT the bytes, and get rejected only at the DB INSERT ‚Äî leaving **orphan R2 bytes no `papic_photos` row accounts for** (never counted, never swept). R2 storage is Setnayan's only marginal per-couple cost, so leaked bytes are the one real cost leak. The cap check was also a non-atomic count-then-insert (a concurrency race).

**The cap is now enforced at the RECORD layer, atomically, and the presign refuses over-cap URLs so no unaccounted bytes can reach R2.**

- **`supabase/migrations/20270222212676_papic_sampler_record_layer_cap_and_capacity_guard.sql`** (new ¬∑ APPLIED to prod via MCP `execute_sql` + ledger row; functions verified present) ‚Äî two SECURITY DEFINER fns (`search_path = public, extensions`):
  - `papic_sampler_insert_capture(seat, kind, r2_key, poster, expires_at)` ‚Äî the **atomic record-layer reservation**. Takes a `SELECT ‚Ä¶ FOR UPDATE` lock on the seat, counts CURRENT (`superseded_at IS NULL`) captures of that kind, and INSERTs the `papic_photos` row **only when under cap** ‚Äî all in one transaction. Concurrent callers serialize on the seat lock, so the (cap+1)th INSERT is impossible. The DB row IS the cap. Returns `{ok, photo_id}` / `{ok:false, error:'sampler_photo_cap'|'sampler_clip_cap'|‚Ä¶}`. Touches `is_free_sampler = TRUE` seats only; a paid/normal seat is refused (`not_sampler`) so it can't be mis-routed.
  - `papic_sampler_remaining(seat, kind)` ‚Äî a cheap capacity probe; returns remaining slots (0 = at cap), or a large sentinel for paid/unknown seats (uncapped from this probe).
- **`apps/web/app/papic/actions.ts`** ‚Äî `recordSeatCapture` now routes sampler inserts through `papic_sampler_insert_capture` instead of the old non-atomic count-then-insert. Paid/normal seats keep the existing direct insert unchanged. Graceful pre-migration fallback (RPC absent ‚Üí 42883/PGRST202 ‚Üí old direct insert) so a not-yet-migrated DB still records captures.
- **`apps/web/app/api/upload/route.ts`** ‚Äî the presign route's papic-seat branch now, for `is_free_sampler` seats, (1) calls `papic_sampler_remaining` and **refuses to mint a presigned PUT URL once the seat is at cap** (409 + `code:'sampler_*_cap'`) ‚Äî no URL ‚áí no bytes reach R2 (the leak-prevention; the atomic RPC is the backstop against a presign-time race), and (2) adds a **per-IP rate limit (40/min)** scoped to the sampler path. This is the anon-farming fix: login-free claim mints a FRESH anonymous `user.id` per leaked link, so the pre-existing per-`user.id` limiter (120/min) didn't curb per-event farming; the per-IP limit does. Scoped to sampler seats so a paid DSLR/camera-bridge burst is never throttled. Fail-open on any probe/limiter hiccup (never breaks the live camera).
- **`apps/web/app/papic/seat/[token]/_components/papic-seat-capture.tsx`** ‚Äî `uploadBlob` surfaces the presign cap 409 as the same `sampler_photo_cap`/`sampler_clip_cap` signal `recordSeatCapture` returns, and `capturePhoto`/`finishClip` treat it as the celebratory "all used up" state (not a scary save error). UX of the happy path is unchanged.
- **`apps/web/lib/papic-sampler-cap-core.ts` + `.test.ts`** (new) ‚Äî pure decision twin of the SQL guard so the cap SEMANTICS are unit-tested in `pnpm test:unit` and the app constants ‚Üî SQL can't silently drift.
