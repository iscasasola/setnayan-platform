## 2026-07-16 · fix(security): block non-admin self-writes of users.is_internal (privilege-escalation guard)

The base `user_owns_row` policy on `public.users` is Pattern A (`FOR ALL … USING/WITH CHECK user_id = auth.uid()`), so an authenticated user can UPDATE *any* column of their own row via the raw Supabase REST API — including the privilege-conferring ones. The app never exposes such a write, but the DB permitted it: an attacker could self-set `is_internal` (the §10a owner/internal flag), `account_type = 'admin'` (which makes `is_admin()` return TRUE), or `is_team_member` (comp-grant eligibility).

- `supabase/migrations/20270814328403_guard_users_privilege_columns.sql`: adds a `BEFORE UPDATE` trigger (`guard_users_privilege_columns`) on `public.users` that silently reverts changes to `is_internal` / `is_team_member` (and any escalation of `account_type` to `'admin'`) to their OLD value for any caller that is NOT privileged. Mirrors the house pattern from `20261214000000_guard_pax_finalize_columns.sql`; does NOT touch the load-bearing base policy. "Privileged" = `auth.role()` is NULL (migration/superuser), `'service_role'` (the `createAdminClient` elevated client), or `public.is_admin()`. Every legitimate write to these columns already goes through the service-role client (verified: §10a/§10b grants + promote-to-admin in `admin/approvals` & `admin/users`; `account_type='vendor'` self-heals in `auth/callback` & `open-shop`), and `on_auth_user_created` sets `is_internal` at INSERT time (BEFORE UPDATE never fires on INSERT) — so no existing flow is affected. Normal profile edits (display_name, slug, theme, consent fields, …) pass through untouched because the guarded columns are unchanged, making the revert a no-op.
- `is_creator` is INTENTIONALLY NOT guarded — the product is moving to user-native creators where any user may self-enable it (documented in the migration header; the column does not exist yet).

SPEC IMPACT: None (security hardening — defense-in-depth on top of the existing RLS policy; no product/pricing/schema-shape change).
