## 2026-06-20 · ui(design-system): unify corner radii on the --m-r-* token scale (softer corners · Approach B)

The app had **three competing corner-radius systems**: a designed `--m-r-*` token scale (4/8/14/22/36/999px in `globals.css`) used only ~6 times, Tailwind's *unconfigured* default `borderRadius` scale driving ~4,200 `rounded-*` classes (which disagreed with the tokens — Tailwind `lg=8` vs token `md=14`), and ~304 ad-hoc hardcoded radii (`border-radius:Npx`, `rounded-[Npx]`, inline `borderRadius:N`). That mix is the classic "inconsistent radius" beginner-tell (owner-supplied UI/UX reference). Owner reviewed an A/B preview and chose **B — softer corners**: standardize everything on the `--m-r-*` token scale.

- **`apps/web/tailwind.config.ts`** — `theme.extend.borderRadius` now maps every `rounded-*` key onto a `var(--m-r-*)` token (`sm→xs · DEFAULT/md→sm · lg/xl→md · 2xl→lg · 3xl→xl · full→full`). One edit re-points all ~4,200 class usages at the token scale; corners get uniformly softer (e.g. buttons `rounded-lg` 8→14, cards `rounded-2xl` 16→22).
- **291 ad-hoc radii codemod'd** across 34 files (scripted, run once, not committed): arbitrary `rounded-[Npx]`→nearest named class, CSS single-value `border-radius:Npx`→`var(--m-r-*)`, inline `borderRadius:N`/`'Npx'`→`'var(--m-r-*)'`. Value-faithful (nearest token; ties round up). Print/doc routes (`**/print/**`, `route.ts`) were **excluded** — they emit standalone HTML/PDF without the app's token CSS.
- **Latent bug fixed:** 4 sites (`manpower/post-gig-drawer.tsx`, `vendor-dashboard/manpower/gig-card.tsx`) referenced `var(--m-radius-md)`, a name that was **never defined** → those corners silently rendered square. Renamed to the real `--m-r-md`.
- **`apps/web/scripts/lint-radius.mjs`** (new guard, `lint:radius`) + **`.github/workflows/ci.yml`** `lint-radius` job (strict) — fail on any new `rounded-[Npx]` / `border-radius:Npx` / inline `borderRadius:N`. Tree is clean at landing (0 findings); allows `var(--m-r-*)`, `0`, `%`, and multi-value shorthand. Not yet a required check — owner promotes via branch protection (same path the nav-icon guard took).

Verified: radius guard passes strict (0 ad-hoc sites); anomaly scan clean (no empty/undefined tokens, no malformed `var()`/`rounded--`, no leftover `--m-radius-`); locked surfaces value-preserving (`bottom-nav` + `globals.css` pill radii: `999`→`var(--m-r-full)` = identical). tsc/lint/build delegated to CI (auto-merge gates on them).
