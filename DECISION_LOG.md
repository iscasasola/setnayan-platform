| 2026-07-21 | **Live Studio console is scroll-free on both layouts (PR #3442) — root cause was `aspect-video`.** Owner (with screenshot): *"obs should be scroll free. for both desktop and mobile."* **Cause:** the PROGRAM monitor used `aspect-video`, which derives height from **WIDTH** — on a wide desktop column that is a ~1000px monitor before the sources rail is reached, so the page scrolled and the SETNAYAN overlay's own bottom band fell off screen. Aspect-ratio boxes suit a card in a feed, not a viewport-filling console. **Principle:** an operator surface used during an unrepeatable ceremony must fit one screen — a control below the fold is a control you cannot reach, and whatever you scroll past is a camera you can no longer see. **Fix:** new `lib/panood-console-fit.ts` — available height is **MEASURED** from the console's own `getBoundingClientRect().top`, never summed from the shell's chrome (sticky bar + `py-6` + `pb-20` mobile nav), because any hardcoded sum rots the first time one of those changes. Mobile-nav clearance follows **VIEWPORT** width (the nav is `lg:hidden`), deliberately NOT the board/compact layout mode — layout is device-driven and the two are allowed to disagree (a laptop in Compact has no bottom nav). `env(safe-area-inset-bottom)` subtracted; negative values cannot add height. **Floors at 420px rather than crushing** — on a short window with OBS docked, scrolling a little is the honest outcome. Returns **null** on an unusable measurement (SSR / zero viewport / mid-transition) so the console falls back to natural flow instead of collapsing. **Layout consequences:** console root = fixed-height flex column with `min-h-0` throughout (without it flex children refuse to shrink below content size and the page scrolls anyway); program `<video>` switches **`object-cover` → `object-contain`** since the box is no longer 16:9 and cropping would make an operator misjudge their own shot; **compact** = the tab body is the ONE scroll region with PROGRAM + tab bar pinned; **board** = columns scroll internally with the rails capped at half the column so a long moment list can never squeeze out the monitor. 9 new tests (124 total). ⚠️ Live on merge. | repo PR #3442 · `apps/web/lib/panood-console-fit.ts` |
