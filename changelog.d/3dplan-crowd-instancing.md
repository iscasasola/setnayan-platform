## 2026-07-08 · perf(plan3d): instanced seated crowd — collapse the phone-walk draw ceiling

> ⚠ OWNER SIGN-OFF PENDING — this changes the seated-crowd RENDERING PATH (individual articulated meshes → instanced), which brushes the owner-locked "one articulated figure everywhere" direction. The LOOK is preserved pixel-for-pixel; the direction nuance is flagged for sign-off before this ships. No PR opened.

The seated crowd drew ~14 non-instanced meshes PER occupant, and the three seat-plan surfaces mounted one `<Figure pose="sit">` per occupied seat with no cap/LOD/instancing — the phone-first public guest walk hit ≈3.2k color-pass draws + ~250 no-op `useFrame` subscribers at 250 pax (quality 'low' dropped shadow casters + per-frame joints but NOT the draw count or the subscription). Because the sit pose is a CONSTANT (`sitPose()` — no id-variation, no time term), every seated figure shares the identical baked joint transform, so the whole seated crowd collapses to ~14 InstancedMesh draws.

- **`lib/figure-sit-bake.ts` (new, pure + unit-tested)** — the SINGLE SOURCE of the rig proportions, per-mesh leaf offsets/scales, joint-length denominators, and `applyPose`, hoisted out of `kit/figure.tsx` (which now imports them; render output byte-identical). `buildSitBakedLocals()` reads each of the 14 seated body parts' local matrix under the sit pose ONCE; `seatRootMatrix` / `seatedFigureMatrix` compose a seat's figure-root world matrix (the latter reproduces the exact table→seat→nudge nesting the surfaces mount `<Figure>` under). One definition drives both the rendered figure AND the baker, so the instanced crowd can never silently diverge from the individual figure it replaces.
- **`lib/figure-sit-bake.test.ts` (new · 6 tests)** — the pixel-identity proof in math form: baked locals equal an INDEPENDENT reference rig (figure.tsx's nesting re-typed with literal proportions), and `seatRootMatrix`/`seatedFigureMatrix × bakedLocal[part]` reproduces the world position an individual nested-group `<Figure>` lands each part at (≤1e-8). Plus determinism, part-count = 14, and the `setColorAt` tint rule (white ≡ neutral, hex ≡ tint).
- **`kit/instanced-seated-crowd.tsx` (new)** — `<InstancedSeatedCrowd>`: one InstancedMesh per body part (reusing the IDENTICAL geometry buffers now exported from `figure.tsx`) + one shared white `MeshStandardMaterial` at the mannequin roughness/metalness, tint via `setColorAt` (`instanceColor` multiplies over white → pixel-identical to `mannequinMaterial(tint)`). Optional instanced status-ring batch (unlit, DoubleSide, per-instance colour; occupants without one zero-scale out; skipped entirely when nobody has a ring). `key={count}` recreates on occupant-count change; `frustumCulled={false}` + `castShadow` honoring quality, matching `InstancedChairs`/`<Figure>`. NO per-figure `useFrame`.
- **`kit/figure.tsx`** — consumes the shared constants; the per-frame joint loop moved into a `<FigureFrameDriver>` child mounted ONLY while animating, so a STATIC (quality 'low' / reduced-motion) individual figure now holds ZERO `useFrame` subscriptions instead of one no-op subscriber apiece. Geometry buffers + ring placement exported for the instanced path.
- **`app/[slug]/venue/_components/guest-venue-3d.tsx` (WIRED — the goal surface)** — neutral, ringless strangers collapse into ONE room-level `<InstancedSeatedCrowd>`; photo seats (billboard head) + the viewer's own seat (accent + gold ring) stay individual. Draw math at 250 pax: ~3.2k seated-figure color draws + 250 `useFrame` subs → ~14 instanced draws + 0 subs. Pixel-identical because this walk is ALWAYS quality 'low' (static baked pose).

**LOD / cap deliberately NOT built (evaluated, left for owner sign-off):** the goal called for the pixel-identical instancing core only. A seat cap or distance-LOD would change the shipped look (fewer/again-different figures), so those are out of this slice.

**Demo (`plan3d-scene.tsx`) + couple lab (`seating-lab-3d.tsx`) left individual (evaluated):** their seated figures animate an idle sway at quality 'high', so instancing them there would DROP that motion = a look change; pixel-identity holds only for STATIC ('low') seated figures. Correctly wiring them needs a quality-gated split (instance at 'low', individual at 'high') plus the ring batch (already supported) plus interaction hit-proxies — a follow-up once the direction is signed off. The `useFrame`-split already benefits their static figures.
