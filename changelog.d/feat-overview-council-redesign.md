## 2026-07-12 · feat(dashboard): Event Overview council redesign — the "doorstep" band + STATUS→ACT→NAVIGATE order

The couple Event Overview (`app/dashboard/[eventId]/_components/event-dashboard.tsx`) was redesigned per the 2026-07-12 model-council verdict (5 design lenses → 3 adversarial critics → synthesis; see corpus `Event_Overview_Council_Verdict_2026-07-12.md`). The owner's signal — the "Around your event" band "does not expand and collapse" — was read by all five lenses as a **symptom that the tiles are dead teasers, not a request for accordions**. Fixes:

- **Band = flat doorways, no accordion (the core ask).** Every "Around your event" tile is now a **whole-card stretched link** to its already-shipped route (`/hosts`, `/vendors`, `/messages`, `/orders`, `/schedule`) via an `after:absolute after:inset-0` overlay on the card's own affordance link (the article is `relative`); depth is one tap away on the real page, never an in-place drawer. **Content-driven auto-density** (the couple never toggles): a substantive tile renders full rows, an empty/quiet tile collapses to ONE calm, hopeful, directive line — so a fresh event reads as a short reassuring scroll, not five barren walls. Hosts collapses to a single "it's just you so far — invite…" line when solo; the roster shows only when 2+ hosts exist.
- **Honest, event-scoped Conversations count.** The tile now reads THIS event's unread vendor threads via `fetchEventUnreadCounts()` (the grouped `unread_message_threads_by_event()` RPC the launcher already uses) instead of `countUnread()` — the **account-wide** notification count that surfaced onboarding/system noise as a scary "32 unread" on a couple with zero vendors. Copy no longer claims false vendor urgency; the unread badge uses the non-gold `warn` (amber) hue so it survives the Atelier+glass reskin (gold becomes the ambient brand accent).
- **STATUS → ACT → NAVIGATE order.** The Journey rail ("Read your progress") moved **below** the band; the Decisions board now leads the daily job directly after the bento. The hero greeting ("you're in the {stage} stage") preserves the reassurance so no emotional pacing is lost. The bento **Decisions ring is now a jump-anchor** (`#decisions`) to the board — a gauge, not a second list.
- **Event-type breadth (safe fallback).** The wedding-only "N of 21 booked" denominator is suppressed for non-wedding event types (debut/christening/corporate/reunion) — they show a plain "N vendors booked" until their per-type category map ships, so no host ever sees a wedding's "of 21". Per-type plan-group maps are a staged follow-up.
- **Urgent-float + one masking note.** Pending-payment orders lead the services roster so they land in the visible slice; the per-card "never a personal profile" legalese is demoted to ONE quiet global footnote at the band's foot (kept the `/activity` link).

No migration — reuses the shipped `unread_message_threads_by_event()` RPC (already LIVE in prod) and existing tokens/primitives. Deferred (owner-approved council follow-ups): bento shape-honest widgets (Budget mini-donut / Guests segmented RSVP bar), full day-of takeover in `page.tsx`, per-event-type plan-group maps, folding the AI "What's next" rail into the Decisions board.
