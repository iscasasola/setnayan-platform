## 2026-07-05 · fix(vendor): route Most Booked + Experience tier + Top Pick + public star average onto VETTED data

Anti-fraud Phase 1 follow-up (stacks on #2834 · the `couple_trusted` badge + `vendor_trusted_review_stats` matview). Closes the remaining self-dealing holes in the badge engine + public rating display by moving the last two RAW-table trust signals onto the vetted, self-dealing-proof stats that already exist. No new migration — reuses `vendor_public_completed_events_stats` (`20260515020000`) + `vendor_trusted_review_stats` (`20270516500000`).

- **Most Booked + Experience tier → vetted completed-events count.** `lib/vendor-badges.ts` — `fetchCompletedBookingCounts` now reads `vendor_public_completed_events_stats.public_completed_count` (keyed by `vendor_profile_id`) instead of raw `event_vendors`. That view already excludes unlinked/archived/owner/team/internal/self-comp bookings, so a vendor can't inflate the `most_booking` percentile by self-creating "delivered" events. `CompletedBookingCounts` re-keyed on `vendor_profile_id` (same id the view + all callers already pass). The Experience-tier `finalizedBookingCount` chip now reads that same vetted count: on `app/explore/page.tsx` it's sourced from the vetted `bookingCounts` map (was `vendor_activity_stats.finalized_booking_count`), and on `app/v/[slug]/page.tsx` from `vendor_public_completed_events_stats` directly. Fail-soft preserved (errors → empty map / 0, never blocks the grid).
- **Top Pick → trusted review stat.** `lib/vendor-badges.ts` — the `top_pick` score is now `trusted_avg_rating × ln(trusted_review_count + 1)` (was raw `avg_rating_overall`/`review_count`). A vendor with high raw reviews but 0 TRUSTED reviews scores 0 (ln(1)=0) and can never enter the top 5%. `spotlight-awards.ts` needed no change — it already threads trusted stats into `computeVendorBadges`.
- **Public star average + count → trusted.** New `fetchTrustedReviewStats` / `fetchTrustedReviewStatsForMany` in `lib/reviews.ts` read `vendor_trusted_review_stats`. Migrated the displayed AGGREGATE number/stars/count (not the review LIST, not the per-star histogram bars, not pagination) on: `app/v/[slug]/page.tsx` (JSON-LD `aggregateRating`, header chip, sticky-rail rating, `ReviewHeroMetrics` headline), `app/explore/page.tsx` vendor cards (`gatedRating`/`gatedReviewCount`), `app/explore/_components/folder-vendors-section.tsx` cards, `app/explore/compare/page.tsx` compare table, and `app/dashboard/[eventId]/_components/vendor-marketplace-info.tsx` (`ReviewsHero`). All tier-gates (Free hides stars → "new") preserved.
- **`lib/vendor-badges.test.ts`**: new cases proving (1) `most_booking` honors the vetted count map — a vendor absent from it (self-dealt events excluded → 0 vetted) never earns it while a peer with vetted bookings does; a big raw `review_count` on the input doesn't count; and (2) `top_pick` scores on trusted stats — a vendor with high raw reviews but 0 trusted reviews never qualifies, and a trusted vendor outranks a raw-only peer. 985/985 unit tests pass.
