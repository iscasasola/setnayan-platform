## 2026-07-16 · feat(seating): the placement oracle — one pure, testable geometry model (council verdict PR-1)

Root-causes the persisting table overlaps at their source: the geometry. Adds the single placement oracle the whole editor will route through (verdict "one oracle, pure and testable in lib/seating.ts"). Pure functions + exhaustive unit tests; **zero UI change** — shipped behavior is untouched (this is the lib-only foundation; the editor/actions wiring lands in PR-2).

New pure helpers in `apps/web/lib/seating.ts` (names locked by the verdict):

- `obbOf(pose)` → rotation-aware oriented footprint. Round = circle; banquet / sweetheart / family-head = a single OBB; serpentine = a **3-OBB decomposition of the arc** (a convex hull would overestimate the concave inner edge — the 3-OBB hugs the curve and leaves the pocket empty). Mirrors `tableGeometry()`'s chair-inclusive footprint exactly.
- `footprintsOverlap(A, B, gap)` → SAT narrow-phase (OBB↔OBB, OBB↔circle closest-point, circle↔circle) with a circumscribed-circle **broad-phase prefilter**; returns penetration depth (0 when clear). Fixes both the old AABB false-negative (a 90°-rotated banquet's real height) and false-positive (phantom-block of an actually-clear slot).
- `checkPlacement(pose, world, params)` → `{ valid, violations[] }` — the oracle: a pose vs all non-groupmates + zones, gap ≥ aisle. Grades violations `overlap` (body intersection) vs `tight` (gap shortfall). Sanctioned contact = same `link_group_id` ONLY (the weld model — no distance-only heuristic).
- `penetrationDepth(pose, world)` → scalar for the monotone-escape drag (out always allowed, deeper never).
- `layoutViolations(poses, zones, gapPx)` → full-board O(n²) audit (mount audit + Auto Arrange verification + server checks).
- `legalJoinPose(anchor, mover, tolPx)` + `isLegalJoint(anchor, mover, pxPerMeter)` → the single source of truth for BOTH snapping and join validation; delegates to the existing snap generators (asserted byte-identical). Join is metric now: end-midpoints within `JOIN_TOL_M = 0.05 m` AND rotation on a legal joint angle (±3°). **The X-crossed-tips "joined" bug dies here** — the retired distance-only `SERP_JOIN_TOL_PX = 18` was wider than the enforcement gap at real room scales.
- `solveAutoLayout(input)` → Auto Arrange as a **verified metric solver** over the same oracle: keeps the stage-out centre-out seed, but every slot must pass `checkPlacement` vs everything placed so far (alternate-slot scan, then next row); the keep-stacking fallback and one-shot zone push are gone. Link groups collapse to rigid super-elements (never split). Metric gaps (slot = aisle, row = aisle + 0.3 m service). Returns `{ placed, unplaced[], altPlacedAtFloor }` — the honest overflow tray plus a real "at 0.6 m it fits N" alternate from a second solver pass. Structurally incapable of returning what drag forbids.

Tests: `apps/web/lib/seating-oracle.test.ts` (23 cases) covers the verdict § 8 list — SAT rotated-banquet overlap the AABB missed + no phantom-block, X-crossed serpentine tips rejected, +104° continuation + 180° S-bend accepted, welded-seam exemption, round kiss any-angle, save/load float-drift round-trip, weld-ghost third-party refusal, monotone-escape monotonicity, a solver property test (random rooms → `layoutViolations === []` always, no coordinate double-book, links never split), and the 0.9 m→36 px / 50 m→14 px scale identities. Full suite 1850 pass; typecheck + lint clean.
