## 2026-06-15 · feat(mobile): disable browser zoom app-wide (native feel) — seat-plan canvas excepted

Owner directive: *"we do not want this to zoom. we want zoom disabled on the whole app except for the seat-plan touch/drag area."* Makes the web/PWA/Capacitor surfaces behave like a native app — no accidental pinch-zoom, no input focus-zoom — while the one genuinely zoomable surface (the seat-plan editor canvas) keeps its gestures.

- **`apps/web/app/layout.tsx`** — viewport `maximumScale: 5` → `maximumScale: 1` + `userScalable: false`. This disables Android Chrome pinch-zoom and the iOS **input focus-zoom** (the auto-zoom on tapping a field). Mounted `<ZoomGuard />` in the body. (Theme-color stays at main's reconciled `#FBFBFA`.)
- **`apps/web/app/_components/zoom-guard.tsx`** (new) — client component. iOS Safari deliberately **ignores** `user-scalable=no` / `maximum-scale` for the deliberate pinch gesture (accessibility), so the viewport meta alone can't stop pinch on iOS. ZoomGuard prevents the WebKit `gesturestart` / `gesturechange` / `gestureend` events at the document level — which disables pinch-zoom **without** affecting scrolling (gesture\* events fire only for pinch/rotate). Any element marked `[data-allow-zoom]` (or nested in one) is skipped. Scope is touch pinch-zoom only; desktop keyboard (⌘/Ctrl +/-/0) and trackpad (ctrl+wheel) browser zoom are intentionally left alone.
- **`apps/web/app/dashboard/[eventId]/seating/_components/seating-editor.tsx`** — added `data-allow-zoom` to the seat-plan canvas (the `ref={canvasRef}` element). The editor runs its own pointer-event pan/pinch on a `touch-none` canvas, so it must receive raw two-finger gestures; this opts it out of the global suppression.
- **`apps/web/app/globals.css`** — pinch-zoom comment block updated (now disabled app-wide); **also added `!important` to the 16px input floor** under `@media (pointer: coarse)` so it beats inline `style={{ fontSize: 13 }}` on the `/login`, `/signup`, `/forgot-password` `FormField` inputs. That fixes the iOS focus-zoom on auth inputs (*"when I clicked on the text box on email login, it zoomed in. After login, it stayed zoomed."*). An inline declaration outranks every selector-based rule — layered or unlayered — so only `!important` lifts those sub-16px fields to the floor on touch devices; verified no input sets an inline font-size **above** 16, so the rule only ever raises, never shrinks. The `touch-action: manipulation` + `overscroll-behavior: none` hardening is unchanged.

⚠ **Accessibility trade-off (owner-acknowledged):** removes WCAG 1.4.4 browser pinch-zoom by explicit owner decision for a native-app feel. OS-level zoom (iOS/Android Settings → Accessibility → Zoom) remains the user fallback.

SPEC IMPACT: None (mobile UX/CSS behavior; no schema/pricing/feature-scope change). Native-app zoom policy logged to corpus `DECISION_LOG.md`.
