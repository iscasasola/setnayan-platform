## 2026-07-16 · feat(vendor): honor a public "hide my prices" setting (opt-in to hide, default show)
SPEC IMPACT: Realizes Social_Share_Settings_Council_Verdict_2026-07-16.md #6 via Option A (invert to hide_prices_publicly) to avoid a marketplace-wide blackout — the old show_prices_publicly opt-in couldn't represent "explicitly false". Note in DECISION_LOG.

Investigation found the collected `show_prices_publicly` flag never gated package/price rendering — it only skipped itself from the public Details list. It is an opt-IN checkbox ("Show prices publicly") defaulting unchecked, stored inside `vendor_service_attributes.attribute_payload` (JSONB), and the vendor form can only ever persist `true` (checked) or an ABSENT key (unchecked/untouched — the checkbox omits the key). An explicit `false` is unrepresentable, so honoring "hide when false" would have blanked prices for every vendor who never touched the box == the entire marketplace.

Option A (owner-approved 2026-07-16) inverts the semantics:
- Migration `20270808300000_vendor_hide_prices_publicly.sql` adds a NEW opt-in-to-hide key `hide_prices_publicly` (boolean, default false = SHOW = today's behavior) to the shared `pricing_signal` attribute group, and marks the old `show_prices_publicly` `retired: true` so it drops out of the vendor form (kept for any vendor who already answered it, per the 0044 never-orphan contract). No data migration — absent/false == show, unchanged for every existing vendor; only a vendor who affirmatively checks the box is hidden.
- `/v/[slug]` now reads the flag (business-wide: true if set on any of the vendor's pricing sections; fails OPEN to "show" on any read error) and suppresses the peso figures on all four price surfaces: per-service "from ₱X" (incl. per-pax/per-hour detail, discount badge, inclusion worth, transport fee), the Packages `PackageCard` total + consumable budget, and the JSON-LD `makesOffer`/`priceRange` (structured data must not leak a price the visible page hides). Package/service names + inclusions stay visible — only amounts disappear.
- Both `show_prices_publicly` and `hide_prices_publicly` are kept in `DETAIL_SKIP_KEYS` so neither shows in the Details list.

Default-safety decision: opt-in-to-hide with default = show preserves current public behavior for all ~existing vendors (no backfill, no blackout); the setting now has real effect only when a vendor deliberately turns it on. The signed-in-host "Customize & lock" modal is intentionally out of scope — it reveals price only inside a deliberate booking action, not passive public display.
