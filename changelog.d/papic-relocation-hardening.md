## 2026-06-25 ¬Ј fix(papic): close two data-loss edges in the samplerвЖТpermanent relocation (pre-lifecycle-rule)

Both are fixed here; they MUST land before that rule ships.

1. **Phase-2 expiry-clear defeated the phase-1 copy-failure fail-safe.** `makeSamplerPermanent` relocates per-row with a deliberate fail-safe (if any `r2Copy` fails, the row is left untouched + its expiry intact, to retry later) вАФ but the unconditional phase-2 `update({expires_at:null}).eq(event_id).not(expires_at is null)` then cleared expiry on that very row, marking a kept couple's still-ephemeral bytes permanent under `papic-sampler/`: invisible to the expiry-based sweep and doomed under a lifecycle rule. **Fix:** phase-2 now also excludes still-ephemeral rows (`.not('r2_object_key','like','%papic-sampler/%')`). `relocateRowToPermanent` already nulls expiry on its own success/already-permanent paths, so phase-2 only backstops legacy/non-ephemeral rows; a copy-failed row keeps its expiry and is retried.

2. **Late capture-derivative `after()` hook re-stranded thumbnails on a converted row.** The fire-and-forget derivative generator runs off the ORIGINAL ref captured at capture time. If a convert relocated the row + nulled its expiry first, the late hook persisted a `derivatives/papic-sampler/вА¶` display/thumb key onto the now-permanent row вАФ stranded. **Fix (two layers):** (a) `makeSamplerPermanent`'s relocation SELECT now matches a row if ANY of its five key columns is under `papic-sampler/` (not just `r2_object_key`), so a derivative-stranded row is re-picked on the next convert / sweep self-heal (the sweep calls this function for kept events, and the filter ignores expiry); (b) `persistDerivativeRefs` now refuses to persist an ephemeral-prefixed derivative key onto a permanent `papic_photos` row вАФ it drops the key (gallery falls back to the relocated permanent original), leaving the leftover bytes as harmless orphans for the lifecycle rule. (`wall_safe` / face-blur is paid-seat-only, so the sampler only ever strands `display`/`thumb`.)
