## Maintenance rule

When adding a new API route:

1. Identify the consumer **before** shipping (UI fetch, webhook source,
   cron runner, monitoring probe).
2. Add a row to this file in the matching section.
3. If consumer is "TODO" (e.g., cron route awaiting owner-side wire),
   document that explicitly in the route file header comment AND
   in this file.
4. Routes with zero callers and no documented future consumer are
   removed in the next orphan-prevention sweep.
