## 2026-07-17 · fix(seating-3d): drop never persists an oracle-rejected pose — monotone escape anchors to the drag-START depth, not the running one

Owner falsified #3349 live (2026-07-17): "I dragged the round table on top of the other … they just overlap. but both can still be clicked and picked up." #3349 "proved" the live 3D drag couldn't create a round-vs-round overlap — but it tested `checkPlacement` **directly** (the ORACLE), never the actual pointer PIPELINE. The oracle is correct; the WIRING leaked.

**The leak — `seating-lab-3d.tsx` `onFloorMove`, the monotone-escape branch.** The escape rule (a violating table may move only to a pose whose body penetration doesn't grow past ε = 2 cm) was judged every frame against the **running** pose: `curDepth = penetrationDepth(cur)` recomputed each frame from the live drag position. With a running reference each frame permits `depth ≤ prevDepth + ε`, but `prevDepth` grows every frame, so the ceiling ratchets inward. Trigger: a table that STARTS within the 0.6 m walkway of a neighbour but bodies-separated (`curValid=false`, `curDepth=0` — the normal spacing of a real reception, and every legacy `layoutViolations` row) — it closes the whole walkway band freely (body depth stays 0), reaches contact, then a **slow continuous** drag drives it 2 cm deeper per frame straight through the neighbour (the figure-8). The 3D lab feeds a continuous raycast, so sub-ε creep is possible; the 2D editor masks the identical running-reference code because it grid-snaps the pointer to 0.5 m steps (a 0.5 m jump can't stay within ε). `commitDrag` then persisted the ratcheted pose with **no single-table re-check** (it trusted `onFloorMove`).

**Fix — anchor the escape baseline to the pose the drag STARTED from, captured once and held for the whole gesture.** Extracted the decision core into pure, exported, tested helpers in `lib/seating.ts`:

- `dragEscapeBaseline(pose, world, params)` → `{ startValid, startDepth }`, captured on the first frame (dragRef still holds the stored start pose).
- `escapeAccepts(pose, world, params, base, epsM)` — a table that started clean must stay fully valid (refuse ALL invalid); one that started violating may rest only at `penetrationDepth ≤ startDepth + ε`.
- `resolveDragStep(poseFor, desired, cur, world, params, base, epsM)` — desired → slide-X → slide-Y → hold, judged against the fixed baseline.

`onFloorMove` now calls `resolveDragStep` with a `dragBaselineRef` captured once per drag (reset on pointer-down + commit). Total body penetration is bounded by `startDepth + ε` for the ENTIRE drag: a clean table can never enter any overlap; a legacy overlap can still be dragged APART to heal but never deeper. `commitDrag` adds a belt-and-suspenders RELEASE GUARD for the single-table path: re-judge the exact (wall-clamped) drop pose against the drag-start baseline via `escapeAccepts`, and if it somehow worsened, settle to the last fully-valid pose seen during the drag (`lastValidPctRef`) — so a release can never persist a pose the oracle rejects. Snapped serpentine joins (a sanctioned touch) and the linked-unit path (already oracle-gated at commit) are untouched. Red-ring visual feedback (`dragValidRef`) unchanged.

**Service-worker stale-JS hedge — verified, no change needed.** `public/sw.js` already network-firsts app bundles across deploys by construction: cache namespaces are stamped per commit SHA (`setnayan-static-${VERSION}`, `stamp-sw.mjs` ← `VERCEL_GIT_COMMIT_SHA`), `activate` evicts every non-current cache + `clients.claim()`, and app navigations (dashboard/seating/lab) are deliberately NOT intercepted (native network → always-fresh HTML referencing content-hashed chunk URLs). The `staleWhileRevalidate` on `STATIC_CACHE` is scoped within a single immutable per-deploy namespace, so the seating/lab route cannot be served stale JS after a deploy. Day-of guest `/[slug]` offline guarantee left intact.
