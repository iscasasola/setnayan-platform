## 2026-07-19 · feat(mobile): disable browser zoom app-wide (native feel) — seat-plan canvas excepted

Owner directive (2026-06-15): *"we do not want this to zoom. we want zoom disabled on the whole app except for the seat-plan touch/drag area."* Makes the web/PWA/Capacitor surfaces behave like a native app — no accidental pinch-zoom, no input focus-zoom — while the one genuinely zoomable surface (the seat-plan editor canvas) keeps its gestures.

**Mechanism note (2026-07-19 repair):** the first cut used the viewport meta (`maximumScale: 1` + `userScalable: false`). That deterministically failed the required Lighthouse check — the `meta-viewport` accessibility audit (weight 10) drops every page to 0.89 < 0.9. Reworked to achieve the identical behavior without the audited meta; the viewport stays at main's `maximumScale: 5`.

- **`apps/web/app/globals.css`** — root `touch-action` hardened from `manipulation` → `pan-x pan-y`: still kills double-tap-zoom + the 300ms delay, now ALSO blocks Android/Chromium pinch-zoom (panning/scrolling untouched, and un-audited by Lighthouse). Also added `!important` to the 16px input floor under `@media (pointer: coarse)` so it beats inline `style={{ fontSize: 13 }}` on the `/login`, `/signup`, `/forgot-password` `FormField` inputs — that (not the meta) is what stops the iOS **input focus-zoom** (*"when I clicked on the text box on email login, it zoomed in. After login, it stayed zoomed."*); iOS only auto-zooms fields whose computed font-size < 16px. Verified no input sets an inline font-size above 16, so the rule only ever raises, never shrinks. `overscroll-behavior: none` hardening unchanged.
- **`apps/web/app/_components/zoom-guard.tsx`** (new) — client guard that preventDefaults WebKit `gesturestart` / `gesturechange` / `gestureend` at the document level. iOS Safari deliberately **ignores** both the viewport meta and root `touch-action` for the deliberate pinch gesture (accessibility), so this is the only reliable iOS lever — and it disables pinch-zoom **without** affecting scrolling (gesture\* events fire only for pinch/rotate). Any element marked `[data-allow-zoom]` (or nested in one) is skipped. Scope is touch pinch-zoom only; desktop keyboard (⌘/Ctrl +/-/0) and trackpad (ctrl+wheel) browser zoom are intentionally left alone.
- **`apps/web/app/layout.tsx`** — mounts `<ZoomGuard />`; viewport meta left Lighthouse-clean at `maximumScale: 5` (comment documents why the zoom-disable does NOT live in the meta). (Theme-color stays at main's reconciled `#FBFBFA`.)
- **`apps/web/app/dashboard/[eventId]/seating/_components/seating-editor.tsx`** — added `data-allow-zoom` to the seat-plan canvas (the `ref={canvasRef}` element) so ZoomGuard skips it. The editor runs its own pointer-event pan/pinch on a `touch-none` canvas, so the root `touch-action` restriction never applied to it — the exception holds on both layers.

⚠ **Accessibility trade-off (owner-acknowledged):** removes WCAG 1.4.4 browser pinch-zoom by explicit owner decision for a native-app feel. OS-level zoom (iOS/Android Settings → Accessibility → Zoom) remains the user fallback.

SPEC IMPACT: None (mobile UX/CSS behavior; no schema/pricing/feature-scope change). Native-app zoom policy logged to corpus `DECISION_LOG.md`.
