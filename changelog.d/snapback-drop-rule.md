## 2026-07-17 · feat(seating): invalid drop is no drop — snap-back rule in 2D + 3D (owner-locked)

Owner directive 2026-07-17 (`Seat_Plan_2D3D_Alignment_Directive_2026-07-15.md` · "Drop rule"): *"we want the tables to be undroppable instead when overlap."* An **invalid drop is NO drop** — releasing a dragged element over an oracle-rejected pose returns it (animated snap-back) to its **drag-START** pose. No settling at an intermediate "last-valid" spot. This **supersedes the monotone-escape COMMIT semantics** shipped the same day in #3358 (settle-to-last-valid) and unifies the rule across BOTH projections.

**One commit rule, one shared helper (`lib/seating.ts`).** New pure, exported, tested `dropAccepted(moved, others, zones, params)`: a drop is accepted iff EVERY moved pose (a single table, or every member of a welded/connective unit translated as a rigid body) is oracle-valid (`checkPlacement`, chair-inclusive, zones, walkway) against everything else — where "everything else" includes the OTHER moved members, so a unit's own legal joints are recognised (`atLegalJoint`) and never self-collide, and a connective SNAP pose is valid by construction. Both editors call this same helper.

**In-drag = free follow + per-frame warning; enforcement = commit.**
- **3D lab (`seating-lab-3d.tsx`).** `onFloorMove` now follows the pointer 1:1 (the monotone-escape resolver is removed from the pipeline); `dragValidRef` is set from a plain per-frame `checkPlacement` so the ground ring goes gold (valid) / warm-red (invalid). `commitDrag` validates the release through `dropAccepted`: valid → `setTables` + persist as before; invalid → NO persist / NO `setTables`, so the mesh's existing `useFrame` lerp eases the table straight back to `home` (its drag-start pose, still in React state) — the animated snap-back, instant under reduced motion. Same for a linked unit (returns as a unit). The `dragEscapeBaseline` / `escapeAccepts` / `resolveDragStep` imports + `dragBaselineRef` / `lastValidPctRef` refs + the settle-to-last-valid RELEASE GUARD are all deleted.
- **2D editor (`seating-editor.tsx`).** `onCanvasPointerMove` free-follows (the running-depth escape block at ~2604/2612 — the latent 2D ratchet — is replaced by the same shared rule) and updates a `dragInvalid` flag that warm-red-rings the dragged table. The drag-START pose(s) of the moved unit are captured once per gesture (`dragStartRef`, the #3358 baseline pattern extended to carry full poses). `onCanvasPointerUp` validates the release through `dropAccepted`; invalid → restore the start pose(s) (a ~280 ms kit-ease `left/top` bounce, instant under `prefers-reduced-motion`), keep selection, brief non-blocking notice ("No room there — needs {aisle} m clear…"), nothing marked dirty. A WELD snap stays a sanctioned join and drops normally.

Legacy healing preserved BY CONSTRUCTION: the drag-start pose is the violating table's own spot, so dragging OUT to a valid pose sticks and any invalid release just returns it — no table can ever get MORE stuck.

Tests — pipeline level (the standard that finally caught the #3358 leak). Extended `lib/seating-3d-drop-pipeline.test.ts` (§D, 5 cases) + new twin `lib/seating-2d-drop-pipeline.test.ts` (pixel space → 2D/3D parity), each over the exact `dropAccepted` commit: (a) drop over another table → persisted pose == drag-start (write equals origin); (b) valid release persists the drop; (c) a legacy overlap dragged OUT to a valid pose persists (healing); (d) a legacy overlap dropped at ANOTHER invalid spot returns to ITS start; (e) a welded serpentine group (legal joint via `legalJoinPose`) with an invalid release returns as a UNIT.
