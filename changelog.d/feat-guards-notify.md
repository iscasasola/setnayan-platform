## 2026-07-09 · feat(setnayan-ai): guards notify — the trigger engine finally taps shoulders (+ 2 gate/webhook security fixes)

The Setnayan AI trigger engine (`runTriggers → applyRestraint`) was dormant: its only consumer was the pull-only account-page digest, the `NotificationType` union had no AI/guard member (guard notifications were structurally impossible), and nothing ever invoked it. This PR is the owner-greenlit "make guards notify" build (previously held as PR-5), following `Setnayan_AI_Realtime_Notifications_2026-07-02.md`.

- **Migration `20270527224949_setnayan_ai_guard_notifications.sql`** — two new `notification_type` enum values (`ai_payment_due`, `ai_guard_alert`) + `setnayan_ai_guard_log`, the persistent dedup/cooldown ledger the spec's restraint engine requires (RLS at CREATE TABLE: couple read-own via `current_event_ids()`, `is_admin()` read, service-role-only writes; no public_id — internal plumbing).
- **Snapshot adapter widened** (`lib/setnayan-ai-snapshot.ts`) — beyond the budget-only money floor: GRD-01 payments now settle per-line via `event_vendor_payments.line_item_id`; GRD-05 budget switches to the Overview/progress committed formula (paid+fulfilled orders + contracted-or-better vendors vs `estimated_budget_centavos`, pending = pending_payment orders); GRD-02 statutory sourced from the `event_paperwork` pipeline + `completeByDate` math; SEC-04 vendor-quiet sourced from pending `chat_threads` with the vendor NAME deliberately masked to a category label (accept-gate: pre-accept threads never leak a name). `priceChanges`/`contracts`/`shortlist`/`dateClusters` stay honestly EMPTY with comments naming the missing sources (price-change log, decision-window model, shortlist age history, availability log).
- **Emission planner + sweep** (`lib/setnayan-ai-guard-plan.ts` pure + tested, `lib/setnayan-ai-notify.ts` server) — guard-category-only, 7-day persistent cooldown, ≤3 per sweep, GRD-01 → email-allowlisted `ai_payment_due`, every other guard in-app-only `ai_guard_alert`; emits to all host members via `emitNotification` (branded email infra reused, no new email system). GRD-01 also stamps a **Resend `scheduledAt` day-before reminder email** (09:00 Asia/Manila, once per due date via a `#d1` ledger row) — the spec's cron-free answer for coverage between app visits.
- **Invocation** — cron-free lazy sweep: `after(() => sweepGuardNotifications(eventId))` in the event dashboard layout (house pattern: `sweepExpiredConcierge`), internally throttled to once/event/6h via the `__sweep__` ledger row (common case = 1 cheap query, post-response).
- **Gating** — `isSetnayanAiActiveForUser` with the DB-first paywall/per-user flags, host-subscription fan-out AND the per-event pricing flag threaded. No AI → no guard notifications.
- **FIX (entitlement)** — `eventOwnsSetnayanAi` no longer gates the 28-day window check on `perEventPricingEnabled` (no read gate in the app threads that flag → a lapsed ₱799 window could NEVER lock once the flag flips). The stored window is now authoritative; windowless rows (every pre-per-event buyer) are byte-identical. Tests updated + regression-locked.
- **FIX (security)** — `/api/notify` now fails CLOSED when `NOTIFY_WEBHOOK_SECRET` is unset (was fail-open: one missing env var silently accepted unauthenticated internet traffic). Extracted to `lib/webhook-secret-auth.ts` + unit-locked.
